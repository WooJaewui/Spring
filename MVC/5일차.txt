 # QueryString 입력.

설명.
queryString의 변수와 같은 이름과 String타입으로 Controller에 매개변수로 선언하면
FrontController에서 알아서 값을 입력해준다.

사용 방법1.
@RequestMapping( "/url" )
public void list( HttpServletRequest request ) { 
	request.getParameter( "queryString" ) ;
}

사용 방법2.
@RequestMapping( "/url" )
public void list( String queryString ) { 
	queryString을 사용한 로직 ;
}



 # QueryString 변수명과 기본값 처리.

@RequestParam( "queryString" ).
QueryString을 메소드의 매개변수로 이용할 때 내가 원하는 변수명으로 사용하는 방법이다.

사용방법.
@RequestMapping( "/url" )
public void list( @RequestParam( name="queryString" defaultValue="10" ) String variableName ) {
	로직 ;
}

추가설명.
name 속성을 통해 파라미터 이름을 설정하여 변수의 값을 얻을 수 있다.
defaultValue를 통해 queryString이 없을때 기본값을 설정할 수 있다.
QueryString은 String으로만 올 수 있는데 int 변수이름을 통해 int 등 다른 타입으로 받을 수 있다.



 # @RequestParam Optional 속성.

required 속성.
required = boolean 값을 설정하여 parameter의 존재여부에 대해 검사한다.
true이면 queryString이 없을때 에러가 발생하고,
false이면 queryString이 없을때 null값을 반환한다.

value.
name과 같은 역할을 한다.
