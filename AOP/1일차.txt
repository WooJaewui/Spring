 # AOP이란?

AOP Aspect Oriented Programming.
관점 지향 프로그래밍.
사용자 관점, 개발자 관점, 운영자 관점 등 다양한 관점으로 프로그래밍을 하는 것이다.
OOP보다 좀 더 큰 개념이다.

Primary( Core ) 
사용자의 관점 주업무를 의미한다.

Core Concern
주업무만 구현하는 소스코드.

Cross-cutting Concern.
주업무 이외에 사이드 업무 소스코드.
주업무 이외에 사이드 업무는 맨위나 맨아래 쪽에 위치하는데, 필요에 따라 컷팅해서 사용하는 방식.



 # AOP 자바 코드 이해하기.

Proxy.
Cross-cutting Concern을 구현하고, Proxy를 통해 주업무 클래스를 호출하여 사용할 수 있다.



 # 순수 자바로 AOP 구현해보기.

순서.
주업무 클래스 생성.
Cross-cutting Concern을 구현할 proxy 생성.
InvocationHandler 인터페이스에 invoke method 안에 사이드 업무를 오버라이딩한다.

예시.
Exam exam = new NewlecExam( 1,1,1,1 ) ;
Exam proxy = ( Exam ) Proxy.newProxyInstance( NewlecExam.class.getClassLoader( ), 
			new Class[ ] { Exam.class },
			new InvocationHandler( ) { 
				
				@override
				public Object invoke( Object proxy, Method method, Object[ ] args ) 
								throws Throwable { 

					실행할 사이드 업무 ;
					Object result = method.invoke( exam, args ) ;
					return result ;
				}
			} ) ;
proxy.메소드이름( ) ;

설명.
실제 클래스를 Proxy를 통해 가짜로 만들어서 메소드이름으로 호출하여 사용할 수 있는 방식이다.
Proxy.newProxyInstance( 실제객체.class.getClassLoader( ), 
		       실제객체가 구현한 인터페이스 배열,
		       InvocationHandler(  ) {
		         	invoke( Object proxy, Method method, Object[ ] args ) {
			method.invoke( 실제객체를 담은 변수, args ) ;
			}
		       } 
proxy.메소드이름( ) : 실제 객체의 메소드를 가짜 proxy로 호출해서 사용.



 # 스프링으로 AOP 구현해보기.

spring의 보조업무 종류.
Before Advice : 앞에만 생성하는 보조업무
After returnning Advice : 뒤에만 생성하는 보조업무.
After throwing Advice : 뒤에서 에러를 처리하는 보조업무.
Around Advice : 앞 뒤 모두 생성하는 보조업무.
