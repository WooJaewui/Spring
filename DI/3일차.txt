 # Dependency를 직접 injection 하기.

직접 injection.
다형성을 사용해서 A라는 인터페이스에 B 또는 C를 선택적으로 사용하는 것을 의미한다.
A a = new B( ) ;
A a = new C( ) ; 

spring을 통한 DI Injection.
new B( )와 new C( ) 부분을 스프링을 통해 사용함으로써 소스코드 변화없이 설정할 수 있다.



 # 스프링 DI 설정을 위해 이클립스 플러그인 설치하기.

지시서. 
DI injection을 어떻게 사용할 것인지 설명한 문서.
XML 또는 annotation으로 작성한다.

플러그인 설명.
지시서 xml파일의 기본적으로 들어가야 되는 내용을 대신 작성해줌. <beans ~~~~~~~~~~ >

플러그인 설치하기.
Help -> Eclipse Market -> spring Tools 3 검색.

플러그인 사용하기.
new -> other -> spring -> Spring Bean Configuration File.



 # 스프링 DI 지시서 작성하기. ( Spring Bean Configuration )

지시서 만들기.
new -> other -> spring -> spring Bean Configuration File -> 파일이름.

클래스 작성하기.
<beans ~~~~~~ >
	<bean id="변수이름" class="패키지이름.클래스이름" />
</beans>

클래스의 메소드 작성.
<beans ~~~~>
	<bean id="변수이름" class="패키지이름.클래스이름">
		<property name="속성이름" value="실제값일 경우" ref="참조형일 경우" />
	</bean>
</beans>

주의사항.
bean의 class에서 패키지는 혼동을 줄이기 위해 사용, 클래스이름이 유일하면 클래스이름만 사용가능.
property의 name은 setName일 경우 name으로 사용한다. ( 약속 )
value와 ref는 변수의 기본형과 참조형을 의미한다.
